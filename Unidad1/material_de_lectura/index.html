Unidad 1 · Introducción, instalación y entorno de trabajo

Curso de Software Estadístico · FACEN–UNA

1. ¿Por qué necesitamos un software estadístico?

En la práctica profesional actual resulta prácticamente imprescindible utilizar un software estadístico. Estos programas permiten gestionar bases de datos de gran tamaño, aplicar procedimientos de análisis estándar y documentar de forma reproducible cada uno de los pasos realizados.

A diferencia de una calculadora o una hoja de cálculo genérica, los softwares estadísticos integran funciones específicas para el tratamiento de datos, la estimación de modelos y la generación de reportes, lo que reduce errores manuales y mejora la trazabilidad de los resultados.

Idea clave: un software estadístico no es solo una calculadora potente, es el eje organizador del flujo de trabajo de análisis de datos.

2. Tipos de software estadístico

Existen dos familias principales de herramientas:

  • Softwares basados en interfaz gráfica (GUI), como SPSS, PSPP o Jamovi, que se controlan principalmente mediante menús y cuadros de diálogo.
  • Softwares basados en código/comandos, como R, Stata o Python con librerías estadísticas, que se utilizan escribiendo secuencias de instrucciones.

En este curso se utilizará una combinación de ambas aproximaciones, enfatizando progresivamente las ventajas de la programación reproducible.

3. Instalación básica de las herramientas

3.1 R y RStudioentorno principal

  1. Acceder al sitio oficial de R (CRAN) y descargar el instalador para el sistema operativo correspondiente.
  2. Instalar R con la configuración por defecto.
  3. Descargar RStudio Desktop (versión gratuita) desde su página oficial.
  4. Verificar que RStudio reconoce la instalación de R y que la consola se encuentra operativa.

3.2 Python y Anaconda

  1. Descargar el instalador de Anaconda para Python desde la página oficial.
  2. Completar la instalación aceptando las opciones recomendadas.
  3. Abrir Anaconda Navigator y lanzar JupyterLab o Jupyter Notebook.
  4. Probar un pequeño script para confirmar que el entorno funciona correctamente.

3.3 SPSS / PSPP / Jamovi

  1. Verificar si la institución dispone de licencia de SPSS u otro software comercial equivalente.
  2. En caso de usar software libre PSPP o Jamovi, descargar el instalador desde la página oficial.
  3. Instalar y comprobar la apertura de la ventana de datos y la ventana de resultados.

4. Elementos visuales de la interfaz

Aunque el diseño específico varía entre programas, la mayoría de los entornos estadísticos comparten una estructura general similar:

  • Barra de menús: agrupa las funciones principales archivo, edición, análisis, gráficos, etc.
  • Barras de herramientas: iconos para acciones frecuentes como guardar, ejecutar o detener código.
  • Ventana de datos: presenta las variables en columnas y las observaciones en filas.
  • Editor de scripts: área donde se escriben y guardan comandos reutilizables.
  • Consola: ejecuta instrucciones individuales y muestra resultados inmediatos.
  • Panel de gráficos y resultados: recoge salidas tabulares y visualizaciones.

5. Flujo de trabajo básico

Antes de ejecutar técnicas avanzadas, es importante interiorizar una secuencia general de trabajo que se repetirá en la mayoría de los proyectos de análisis de datos.

  1. Definir una carpeta de trabajo para el proyecto.
  2. Importar el conjunto de datos, por ejemplo, desde CSV o Excel.
  3. Verificar tipos de variables, nombres y valores atípicos evidentes.
  4. Realizar una primera exploración descriptiva tablas de frecuencia, medias, gráficos simples.
  5. Guardar scripts y resultados relevantes.
  6. Registrar la versión del software utilizada y la fecha del análisis.

6. Buenas prácticas iniciales

  • Mantener datos originales inalterados y trabajar sobre copias derivadas.
  • Documentar las transformaciones realizadas sobre las variables.
  • Utilizar nombres consistentes para objetos, scripts y archivos de salida.
  • Respaldar periódicamente la carpeta del proyecto.
